# deployment/ansible/roles/backup_agent/sample.txt

[데이터 흐름 구조]
1. [ccmall_web > onprem DB]
    : 메인 데이터 저장
- 방식 : FastAPI 애플리케이션이 SQLAlchemy를 통해 onprem PostgreSQL에 직접 저장
- 주기 : 실시간이며, 사용자의 요청 발생 즉시 저장함
- 검증 : 애플리케이션 레벨에서 입력값 검증 및 DB 제약 조건

2. onprem DB > ccmall_rec
    : pg_dump, RPO를 줄이기 위한 주기적 백업 및 동기화
- 방식 : pg_dump 기반 백업
        ccmall_rec 서버에서 APScheduler가 주기적으로 백업 스크립트 실행
        onprem DB dump 생성 후 ccmall_rec DB에 반영
- 주기 : 전체 백업은 하루 1회, 새벽 시간대
        변경 빈도 높은 테이블 -> 20분 주기
- 검증 : dump 파일 생성 여부 및 0 여부
        restore 성공 여부 및 주요 테이블 row count 비교
        최신데이터 timestamp 비교 등


3. ccmall_rec > S3
    : 오래된 데이터 export(장기 보관)
6개월 이상 지난 거래/로그 데이터 export
S3 업로드 성공 후 ccmall_rec에서 삭제
